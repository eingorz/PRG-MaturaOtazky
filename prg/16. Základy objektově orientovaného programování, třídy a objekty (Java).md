# Co je OOP?
- OOP je programovací model (paradigm); v OOP je kód součástí dat (Metody jsou zapouzdřeny v objecktech), což umožňuje přehlednější kód a logiku programu a zároveň snadnější přenos kódu mezi objekty (abstrakce, zapouzdření)

## Základní pojmy OOP

### Třída (class)

- "šablona" pro vytváření objektů (strukturu dat)
- definuje atributy (data) a metody (chování)

```Java
class Auto {
    String znacka;
    int rychlost;

    void jed() {
        System.out.println("Auto jede");
    }
}
```

### Objekt (instance)

- konkrétní vytvořený prvek třídy
- V Javě tvoříme pomocí klíčového slova `new`

```Java
Auto mojeAuto = new Auto();
mojeAuto.znacka = "Škoda";
mojeAuto.jed();
```

### Atribut (vlastnost, field)

- proměnná uvnitř třídy
- popisuje stav objektu (pokud není static)

```Java
String znacka;
int rychlost;
```

### Metoda

- funkce definovaná v třídě
- popisuje chování objektu

```Java
void jed() {
    System.out.println("Auto jede");
}
```

## Základní principy OOP

### Zapouzdření (Encapsulation)

skrytí dat a řízení přístupu k nim

používají se přístupové modifikátory:
- `private`
- `public`
- `protected`

```Java
class Osoba {
    private String jmeno;

    public String getJmeno() {
        return jmeno;
    }

    public void setJmeno(String jmeno) {
        this.jmeno = jmeno;
    }
}
```
Využívá se k:
- ochraně dat
- kontrole změn

### Dědičnost (Inheritance)

- jedna třída sdílí vlastnosti s jinou
- používá se `extends`

```Java
class Zvire {
    void zvuk() {
        System.out.println("Nějaký zvuk");
    }
}

class Pes extends Zvire {
    void zvuk() {
        System.out.println("Haf");
    }
}
```
Používá se k:
- znovupoužitelnosti kódu
- hierarchii tříd

### Polymorfismus

- schopnost objektů chovat se různě
- stejná metoda → různé chování

```Java
Zvire z = new Pes();
z.zvuk(); // zavolá se metoda z třídy Pes
```

### Abstrakce

- zjednodušení složitosti
- zaměření na podstatné vlastnosti

```Java
abstract class Tvar {
    abstract double obsah();
}
```

## Rozhraní (Interface)

- předpis, definuje které metody musí třída implementovat
- kostra, díky implementaci interface lze předpokládat, že třída tyto abstraktní metody definuje

```Java
interface Letajici {
    void let();
}

class Pták implements Letajici {
    public void let() {
        System.out.println("Pták letí");
    }
}
```

Rozdíl od třídy:
- nelze vytvářet instance
- metody jsou (většinou) abstraktní (klíčové slovo `default`)

## Konstruktor

- Speciální metoda pro inicializaci objektu
- Má stejný název jako třída

```Java
class Auto {
    String znacka;

    Auto(String znacka) {
        this.znacka = znacka;
    }
}
```

```Java
Auto a = new Auto("BMW");
```

Na rozdíl např. od C++, Java nemá nic jako "destruktor"

## Přístupové modifikátory

Určují jestli je proměnná/metoda viditelná z kontextu jiného než zevnitř třídy

| Modifikátor | Přístup            |
| ----------- | ------------------ |
| `public`    | odkudkoliv         |
| `private`   | pouze ve třídě     |
| `protected` | ve třídě + potomci |
| (default)   | v rámci balíčku    |

## Přetěžování a přepisování metod

### Přetěžování (Overloading)

Stejné jméno, různé parametry

```Java
int soucet(int a, int b) { return a + b; }
double soucet(double a, double b) { return a + b; }
```

### Přepisování (Overriding)

Redefinice metody v potomkovi

```Java
@Override
void zvuk() {
    System.out.println("Haf");
}
```

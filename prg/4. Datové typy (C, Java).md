### Definice struct a union (s typedef)
Abychom nemuseli při každém vytváření proměnné v C psát klíčové slovo `struct` nebo `union`, definují se tyto strukturované typy nejčastěji rovnou pomocí `typedef`.

```c
// STRUCT
// Normální definice
struct Osoba {
    char jmeno[50];
    int vek;
};
struct Osoba clovek1; // Při deklaraci musíme psát slovo "struct"

// Definice s okamžitým vytvořením instancí
struct Auto {
    char znacka[50];
    int rokVyroby;
} auto1, auto2; // Proměnné auto1 a auto2 jsou vytvořeny rovnou s definicí struktury

// Definice s typedef 
typedef struct {
    char jmeno[50];
    int vek;
    float vyska;
} Student;

Student s1; // Rovnou používáme "Student" jako nový datový typ
s1.vek = 20; // Přístup k prvkům přes tečkovou notaci

// Přístup přes pointer
Student *ptr = &s1;
ptr->vek = 21; // U pointerů na struct se používá šipka (->)


// --- UNION ---
// Definice s typedef
typedef union {
    int celeCislo;
    float desetinneCislo;
    char znak;
} Hodnota;

Hodnota h1;
// U unionu všechny prvky sdílí stejnou paměť! 
// Velikost unionu = velikost největšího prvku (zde int/float zabírají 4 byty)
h1.celeCislo = 42;
h1.desetinneCislo = 3.14; // POZOR: Tímto se přepíše hodnota celeCislo!
```

### typedef
- Vytváří alias (nové jméno) pro existující datový typ
- Zjednodušuje zápis komplexnějších typů a struktur
```c
typedef struct {
    int x;
    int y;
} Bod;

Bod b1; // Místo "struct Bod b1;"
```

## Java
- reference vs primitive
- reference data types:
	- můžou být uživatelsky definované
	- ukládají adresy
	- umožňují několikanásobné proměnné
	- žerou víc paměti
	- pomalejší
- primitive data types:
	- 8 základních typů
	- ukládají data
	- pouze jedna hodnota
	- žerou méně paměti
	- rychlejší


## Základní definice  
  
- **Pole (array)** je **souvislá oblast paměti** obsahující prvky **stejného datového typu**.  
- To co obvykle označujeme za proměnnou typu pole je proměnná typu pointer na nultý prvek pole
- Přístup k jednotlivým prvkům je realizován pomocí **indexu**. (Pointerovou aritmetikou)
- Indexování v C **začíná od 0**.  
- Velikost pole musí být při kompilaci obvykle **známá** (výjimka: VLA v C99).  
  
V paměti jsou prvky pole uloženy **sekvenčně bez mezer**, což umožňuje velmi efektivní přístup.  
  
---  
## Deklarace pole  
  
Syntaxe:  
```c  
typ jmeno_pole[velikost];
```

Příklad:
```c
int arr[5];
```

Tím vznikne pole o **5 prvcích typu `int`**.

---
## Inicializace pole

### Explicitní inicializace
```c
int arr[5] = {1, 2, 3, 4, 5};
```
### Částečná inicializace

Zbývající prvky jsou automaticky nastaveny na **0**.
```c
int arr[5] = {1, 2};
```

Paměť:
`[1, 2, 0, 0, 0]`

---
### Odvození velikosti pole

Velikost pole lze nechat určit kompilátorem.
```c
int arr[] = {1,2,3,4};
```

Velikost pole = **4**.

---
## Přístup k prvkům pole

Prvky se přistupují pomocí **indexu**. Idexovací operátor je pouze syntakticky rozdílný od pointerové aritmetiky a dereference.

`arr[index]` = `*(arr + index)`

Příklad:
```c
int arr[3] = {10,20,30}; 
  
printf("%d", arr[1]); // 20
```

---
## Interní reprezentace pole

Pole je implementováno jako **souvislá paměťová oblast**.

Adresa prvku:
`base_address + index * sizeof(type);`

Například pro `int arr[5]`:
```c
arr[0] -> base  
arr[1] -> base + 4  
arr[2] -> base + 8
```

---
## Pole jako pointer

V mnoha kontextech se název pole **automaticky převádí na pointer na první prvek**.
```c
int arr[5];  
int *p = arr;
```

Platí vztah:
```c
arr[i] == *(arr + i)
```

Příklad:
```c
int arr[3] = {1,2,3};  
  
printf("%d", *(arr + 1)); // 2
```

Důležitý rozdíl:
- `arr` je **konstantní adresa**
- pointer lze měnit

```c
int *p = arr;  
  
p++;    // OK  
arr++;  // chyba
```

---
## Pointerová aritmetika u polí

Pointerová aritmetika umožňuje iteraci přes pole.

```c
int arr[5] = {1,2,3,4,5};  
  
for(int *p = arr; p < arr + 5; p++){  
    printf("%d\n", *p);  
}
```

Posun pointeru respektuje **velikost datového typu**.

---
## Velikost pole

Velikost pole lze zjistit pomocí `sizeof` v kontextech, kde kompilátor zná velikost pole za doby kompilace.

```c
int arr[10];  
  
sizeof(arr)        // velikost pole v bytech  
sizeof(arr[0])     // velikost prvku
```

Počet prvků:
```c
int count = sizeof(arr) / sizeof(arr[0]);
```

Důležité:
- funguje **pouze v místě deklarace pole**
- při předání funkci se pole **degraduje na pointer**

---
## Pole jako parametr funkce

Pole se do funkcí předává **jako pointer na první prvek**.
```c
void printArray(int arr[], int size) {  
    for(int i = 0; i < size; i++)  
        printf("%d ", arr[i]);  
}
```

Ekvivalentní deklarace:
```c
void printArray(int *arr, int size)
```

Proto funkce **nezná velikost pole**, musí se předat samostatně.

---
## Modifikace pole ve funkci

Protože se předává **adresa prvního prvku**, změny se projeví i mimo funkci.

```c
void change(int arr[]){  
    arr[0] = 100;  
}
```

---
## Proměnná velikost pole (VLA)

Od standardu **C99** lze deklarovat pole s velikostí známou až za běhu.
```c
int n;  
scanf("%d", &n);  
  
int arr[n];
```

Omezení:
- pouze **automatická paměť (stack)**
- ne všechny kompilátory plně podporují.
- při tvorbě příliš velkého pole takto dojde k přetečení zásobníku (do haldy, nebo na svůj vlastní začátek) (stack overflow), což by se ovšem dělo i kdybychom pole o stejné velikosti vytvářeli za běhu ve statickém kontextu voláním funkce která toto pole tvoří jako svou lokální proměnnou, takže to není o nic nebezpečnější než např. neohlídaná rekurze

---
## Dynamická pole

Dynamická alokace pomocí `malloc`, `calloc`, `realloc`.
```c
int *arr = malloc(10 * sizeof(int));
```

Alokuje blok paměti na haldě, pokud není možno paměť alokovat, vrací `NULL`

Uvolnění:
```c
free(arr);
```

Bez uvolnění vzniká **memory leak** (semantická chyba úniku paměti).

---
## Přetečení pole (Buffer Overflow)

C **nekontroluje hranice pole**.
```c
int arr[3];  
  
arr[5] = 10; // undefined behavior
```

Může způsobit:
- přepsání paměti
- pád programu (segmentační chyba)
- bezpečnostní zranitelnosti

---
## Optimalizace práce s poli

Díky sekvenčnímu uložení dat jsou pole **cache-friendly**.
Výhody:
- rychlý přístup O(1)
- efektivní iterace
- optimalizace kompilátorem

Nevýhody:
- pevná velikost
- nákladné vkládání/mazání uvnitř

---
## Typické chyby

### Přístup mimo rozsah
```c
int arr[3];  
  
arr[3] = 5; // chyba
```

---
### Ztráta informace o velikosti
```c
void f(int arr[]){  
    sizeof(arr); // vrátí velikost pointeru  
}
```

---
### Špatná práce s dynamickým polem

```c
int *arr = malloc(10 * sizeof(int));  
  
arr = malloc(20 * sizeof(int)); // memory leak
```

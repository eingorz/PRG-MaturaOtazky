-  Deklarace a charakteristika dynamického vícerozměrného pole:
	- Pole pointerů
	- Pointer na pole
	- Pointer na pointer
## Statické
- Víme, jakou velikost bude vícerozměrné pole mít at runtime
```C
int xd[x][y];
// X => radky
// Y => sloupce
```
- po alokaci jsou všechny prvky hned za sebou v paměti, lze používat pointerovou aritmetiku místo přístupu přes \[]\[]
``` C
int xd[2][3];
int *y = (int *)(xd + 3); // 3. prvek v poli
```
- Nutno podotknou, že stejně jako v jednorozměrném poli, "xd" pouze ukazatel na první prvek v poli
## Dynamické
- Pole pointerů
	- Je potřeba předem vědět počet řádků, nikoliv však sloupců
	- V zásobníku naalokujeme pole pointerů (normální proměnná)
	- Na každý pointer v poli dynamicky naalokujeme pole v haldě
```c
int sloupce = 4;
int *pole[3]; // Pole 3 pointerů na zásobníku (počet řádků musí být známý)

for (int i = 0; i < 3; i++) {
    pole[i] = (int *)malloc(sloupce * sizeof(int)); // Alokace sloupců v haldě
}
pole[0][0] = 42; // Přístup k prvkům
```
- Pointer na pointer
	- Není potřeba předem vědět počet sloupců či řádků
	- V zásobníku vytvoříme pouze pointer
	- Na pointer prvně naalokujeme pole pointerů v haldě (počet řádků)
	- Na každý pointer v poli naalokujeme další pole (sloupce)
```c
int radky = 3;
int sloupce = 4;
int **pole; // Pouze pointer na zásobníku

// Alokace pole pointerů (řádky)
pole = (int **)malloc(radky * sizeof(int *));

// Alokace jednotlivých řádků (sloupce)
for (int i = 0; i < radky; i++) {
    pole[i] = (int *)malloc(sloupce * sizeof(int));
}
pole[1][2] = 42; // Přístup k prvkům
```
- Pointer na pole
	- V zásobníku vytvoříme pointer na _datový typ_ pole o XY prvcích
	- Je potřeba předem vědět počet sloupců
```c
int radky = 3;
#define SLOUPCE 4
int (*pole)[SLOUPCE]; // Pointer na pole o 4 prvcích (počet sloupců musí být známý)

// Alokace souvislého bloku paměti pro všechny řádky najednou
pole = malloc(radky * sizeof(*pole));

pole[2][3] = 42; // Přístup k prvkům
```

## Vícerozměrné pole jako parametr funkce

Při předávání vícerozměrného pole do funkce se přístup liší podle toho, zda je pole alokováno staticky nebo dynamicky.

### 1. Rozměry v hranatých závorkách (Statická pole)
Když předáváme statické vícerozměrné pole, **musíme kompilátoru sdělit velikost všech dimenzí kromě té první** (u 2D pole tedy počet sloupců), aby věděl, jak správně počítat offsety v paměti.
- První rozměr může zůstat v hranatých závorkách prázdný `[]`.
- Velikost tohoto prvního rozměru se pak typicky předává jako **další parametr funkce**, aby funkce věděla, kolik prvků má zpracovat.
- Toto pravidlo platí univerzálně pro pole o 3, 4 a více rozměrech – vždy smí zůstat prázdná pouze první závorka, všechny ostatní musí mít definovanou velikost.

```c
// 2D pole: Počet sloupců (4) musí být pevně dán v parametru
void zpracuj2D(int radky, int pole[][4]) {
    for(int i = 0; i < radky; i++) {
        pole[i][0] = 1; // Přístup normálně přes [][]
    }
}

// 3D pole: Musíme pevně uvést 2. a 3. dimenzi
void zpracuj3D(int dim1, int pole[][5][10]) {
    pole[0][2][4] = 1; // Přístup přes [][][]
}

// 4D pole: Musíme pevně uvést 2., 3. a 4. dimenzi
void zpracuj4D(int dim1, int pole[][2][3][4]) {
    pole[0][1][2][3] = 1; // Přístup přes [][][][]
}
```
### 2. Předání přes pointer (Dynamická pole)
Pokud je pole vytvořeno dynamicky jako *pointer na pointer* (pole pointerů), do hranatých závorek se nepíše nic a jako parametr se předává přímo pointer. 
- V tomto případě kompilátor nezná strukturu pole, takže musíme jako **další parametry** předat **počet řádků i počet sloupců** (pokud je funkce oba potřebuje znát).

```c
// Předáváme pointer na pointer a oba rozměry zvlášť
void zpracujDynamicke(int radky, int sloupce, int **pole) {
    for(int i = 0; i < radky; i++) {
        for(int j = 0; j < sloupce; j++) {
            pole[i][j] = 0; 
        }
    }
}
```

Pokud předáváme *pointer na pole* (souvislý blok v haldě), syntaxe parametru vypadá takto:
```c
// Předáváme počet řádků jako parametr, sloupce musí být definovány u pointeru
void zpracujPointerNaPole(int radky, int (*pole)[4]) { ... }
```
